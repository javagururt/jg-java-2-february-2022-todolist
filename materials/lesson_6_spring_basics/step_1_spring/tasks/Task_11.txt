Очень часто в приложениях есть параметры конфигурации,
которые необходимо менять без перекомпиляции кода
самого приложения. Например это может быть включение
или выключение какого нибудь функционала.

Если этот переключатель реализовать только в коде
приложения (например как свойство класса или
константу) то при смене его значения придётся
полностью перекомпилировать весь код приложения.
Полная компиляция и сборка приложения это не быстрый
процесс, который к тому же требует наличие определённых
инструментов (например билд тула Gradle).

Для того, чтобы избежать полной перекомпиляции
кода приложения при смене значения в конфигурации,
саму конфигурацию нужно вынести за пределы кода
приложения и сделать так чтобы при старте приложение
читало параметры конфигурации из внешнего источника
(например файла).

Одним из стандартных вариантов решения этой задачи
являются файлы конфигурации. Такие файлы находятся
за пределами Java кода и их можно менять без полной
перекомпиляции приложения. При старте приложение
читает параметры конфигурации из отдельного файла
и применяет их во время своей работы.

Для изменения значения параметра конфигурации
достаточно просто поменять значение параметра
в файле конфигурации и рестартануть приложение.
При старте приложение прочитает новое значение
параметра из файла конфигурации и применит его.
Полная перекомпиляция кода приложения в этом
случае не нужна.
