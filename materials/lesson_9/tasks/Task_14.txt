В пакете database переименуйте интерфейс Database в BookRepository.
В приложении будет много доменных объектов и для каждого из них
нужны будут методы для работы с базой данных. Добавляя все эти
методы в один интерфейс мы сильно усложним его. Для простоты
управления кодом работы с базами данных принято для каждого
доменного объекта создавать свой класс в котором будут
храниться методы работы с базой данных.
Пример:
Book -> BookRepository
Product -> ProductRepository
и так далее.
