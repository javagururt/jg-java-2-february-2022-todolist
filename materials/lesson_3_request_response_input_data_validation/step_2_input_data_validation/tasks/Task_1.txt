Обрабатывая входные данные введённые пользователем
мы не можем полагаться на их качество. Пользователь
нашей системы это человек, а человеку свойственно
иногда ошибаться. Человек может вместа числа ввести
строку, или наоборот, неправильно указать свой адрес
эдектронной почты (ошибиться в формате) и так далее.

Входные данные система должна обязательно проверять/валидировать
и только потом сохранять их в базу данных или выполнять с ними
какие то операции. Если при валидации обнаружатся критические ошибки,
то система должна сразу останавливать процесс обработки запроса,
формировать ответ со списком ошибок и возвращать его на UI для
показа пользователю.
