Давайте попробуем разобраться, какую информацию об ошибке
валидации бизнес логика должна вернуть на сторону UI для того,
чтобы UI смогла корректно показать в чем ошибка пользователю?

Бизнес логика должна указать название поля, которое пользователь
ввел неправильное или не валидное значение, а так же бизнес логика
должна вернуть описание ошибки, почему она считает введенное значение
не корректным. Пример описания ошибки: Это поле обязательное для ввода!

Для описания одной ошибки валидации создайте класс CoreError
со следующими полями:
    private String field;
    private String message;
с конструктором класса Error (String field, String message)
и get() методами для каждого поля.
