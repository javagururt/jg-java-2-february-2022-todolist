В реальной библиотеке книг может быть очень много.
При поиске не целесообразно, а порой даже не возможно
возвращать на UI все найденные книги.
Предположим во время поиска было найдено 1000 книг.
Конечно бизнес логика может вернуть все 1000 книг на
сторону UI, вопрос что будет делать UI со списком из
1000 книг, ведь показать на экране или на веб странице
сразу 1000 книг будет не разумно. Во первых это огромный
список который пользователю придётся пролистывать очень долго,
во вторых вывод большого количества данных на UI всегда
затруднительно, всё просто начнёт тормозить и пользователю
будет тяжело с этим работать.
Для решения данной задачи был придуман механизм постраничного
просмотра данных, когда пользователю не показывают сразу все
данные, а дают возможность начать работать с первой страницей,
на которой отображают первые 10 или 20 записей. В конце страницы
пользователю дают возможность запросить следующую страницу данных.
Такой постраничный вывод информации не перегружает UI и остальные
уровни приложения, но у пользователя всегда остаётся возможность
запросить следующие страницы.

В этом разделе вам предлагается добавить и реализовать
функционал постраничного просмотра информации к поиску книг.
